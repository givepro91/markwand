- **#1 최근 7일 이미지 탭** — 사용자가 이미지 문서도 최근 활동으로 추적하고 싶어 함. 현재 `RecentDocsPanel` 은 md-only. 단, 섞이면 안 되고 탭/토글로 분리 요구. 기존 패널 구조에 탭 컴포넌트만 얹으면 됨.

## Problem (문제 정의)

### 핵심 문제
사용자 실사용에서 드러난 3개의 작고 독립적인 기능 갭(Mermaid 렌더 품질 · 프로젝트 목록 자동 반영 · RecentDocsPanel 의 이미지 미지원)을 하나의 beta 릴리스로 묶어 최소 범위 변경으로 해소한다.

### MECE 분해

| # | 문제 영역 | 설명 | 영향도 |
|---|----------|------|--------|
| 1 | Mermaid 옵션 | `mermaid.initialize` 가 `startOnLoad`/`theme` 만 지정. `flowchart.htmlLabels` 와 `themeVariables.fontSize` 가 default 라 노드 auto-sizing 이 실제 렌더 폭과 불일치 → 텍스트 잘림 | 중 |
| 2 | 프로젝트 목록 자동 반영 | `refreshKey` 는 수동 새로고침 버튼 1개 경로만 존재. `fs:change` 이벤트는 renderer 에서 `useDocs` 가 doc-level 로만 소비. 프로젝트 레벨 변화(root depth 1~2 의 디렉토리 add/unlink) 는 list invalidate 까지 전달되지 않음 | 높 |
| 3 | RecentDocsPanel md-only | `.filter(...classifyAsset === 'md')` 하드코딩. 이미지는 정렬/표시 양쪽에서 완전히 배제 | 중 |
| 4 | prefs 스키마 확장 | 새 prefs 키 `recentDocsTab: 'docs' \| 'images'` 필요. 타입/기본값/persistence 3곳 맞물림 | 낮 |
| 5 | i18n 리소스 | ko/en 양쪽에 새 탭 라벨·빈 상태 문구 추가 필요 | 낮 |

### 제약 조건
- **범위 가드**: 뷰어 리디자인(#6) · UX 친화 개편(#7) · 하이라이팅(#3) 은 이번 Plan 에 포함하지 않음. 불명확한 스코프 확장 금지.
- **회귀 가드**: 기존 md-only 동작이 기본값으로 유지되어야 함. prefs 없는 사용자는 'docs' 탭이 선택된 상태로 부팅되고 기존 사용자와 동일한 첫 화면.
- **성능 가드**: 프로젝트 목록 자동 싱크가 watcher storm(massive add/unlink)에서 무한 rescan 을 유발하지 않아야 함 → debounce 2초, refreshKey 는 내부적으로 throttle.
- **SSH 범위**: SSH 는 SshPoller 가 이미 30s 주기로 돌고 있어 자동 싱크 범위에서 제외. 로컬만 다룸 (SSH 목록 자동 갱신은 별도 후속 — 본 Plan Known Gap 에 기록).
- **DC-4 유지**: SSH hostVerifier 동작 변경 없음.
- **beta.8 보존**: 기존 prefs (`recentDocsCollapsed`, `language`, `viewLayout` 등) 스키마 호환.

## Solution (해결 방안)

### 선택한 방안

| 스프린트 | 접근 |
|---|---|
| S1 (Mermaid fix) | `mermaid.initialize` 에 `flowchart: { htmlLabels: true, useMaxWidth: true, padding: 8 }` + `themeVariables: { fontSize: '14px' }` 추가. CSS 에서 `.mermaid-block svg { max-width: 100%; height: auto; }` 로 overflow guard. 테마 전환 시 재초기화 경로(`setMermaidTheme`)에도 동일 옵션 주입 |
| S2 (자동 싱크) | main watcher 가 **이미 디렉토리 add/unlink 이벤트를 수신** (chokidar ignored 로직이 "확장자 없는 경로=디렉토리" 통과) → 새 IPC `projects:invalidate` 이벤트를 main 에서 emit. renderer 는 workspaceRoot/depth ≤ 2 범위의 디렉토리 변화를 감지하면 2s debounce 후 `bumpRefreshKey()` 호출. 기존 refreshKey 재스캔 경로 재사용 → App.tsx 수정 최소화 |
| S3 (최근 이미지 탭) | `RecentDocsPanel` 에 `role="tablist"` + 2 탭 버튼. `prefs.recentDocsTab` 으로 마지막 선택 복원. 탭별 `useMemo` 로 분리된 최근 목록 계산. 둘 다 비어야 패널 null. a11y: `aria-controls` + `aria-selected` |

| 수정량 | 2 파일 | 3~4 파일 |

본 Plan 에 포함하지 않는 항목 — 별도 스프린트에서 처리:

- **이슈 #3 하이라이팅/형광펜** — 신규 annotation 데이터 모델 설계 필요. 별도 Plan.

- **이슈 #4 ASCII art 코드블록 개선** — 원본 마크다운을 건드리지 않고 앱이 개선할 수 있는 여지가 제한적. 6번 스프린트에서 묶어 검토.
