# Markwand v0.3.0-beta.4 — DMG 도움말 포맷 교체 (.webloc → .html)

**배포일**: 2026-04-22
**배포 채널**: 내부 베타 (unsigned dmg + ad-hoc signing)
**이전 릴리스**: v0.3.0-beta.3 (2026-04-22)

기능 변경 없음. beta.3 의 `.webloc` 이 macOS Tahoe (26.2) 에서 "문서 콘텐츠를 읽을 수 없습니다" 에러로 파싱 실패하는 문제 해소.

---

## 🚀 설치 빠른 경로 (macOS 15+ / 26+)

1. [Assets](#assets) 에서 본인 Mac 기종의 DMG 다운로드 (Apple Silicon 은 `arm64`)
2. DMG 더블클릭 → 창이 뜸
3. `Markwand` 를 **Applications** 폴더로 드래그
4. Applications 에서 Markwand 더블클릭 → "악성 코드 확인 불가" 경고 → **완료**
5. **DMG 창의 `여기를 먼저 더블클릭.html` 파일을 더블클릭**
   - Safari 가 열리며 안내 페이지 표시
   - 페이지가 자동으로 시스템 설정 → 개인정보 보호 및 보안 패널을 엽니다
   - 안 열리면 페이지 안의 **시스템 설정 열기** 버튼 클릭
6. 스크롤 → `Markwand.app 이(가) 차단되었습니다` 옆 **그래도 열기** → Touch ID
7. Markwand 다시 더블클릭 → **열기** → 끝

---

## 바뀐 것

- `build/설치 도움말.webloc` 삭제
- `build/여기를 먼저 더블클릭.html` 신규:
  - `<meta http-equiv="refresh">` 로 `x-apple.systempreferences:` URL 자동 호출
  - 파일명 자체가 행동 유도 ("여기를 먼저 더블클릭")
  - Safari 페이지에 한국어 설명 + 큰 "시스템 설정 열기" 버튼 (자동 호출 실패 시 수동 fallback)
  - 라이트/다크 모드 자동 대응
- `dmg.contents` 의 경로 갱신

## 왜 .html 인가

`.webloc` 은 Safari 가 생성하는 URL 북마크지만 **형식이 binary plist / XML plist 둘 다 지원**되고, macOS Tahoe 부터는 XML plist 로 수동 작성한 `.webloc` 을 Finder 가 거부할 때가 있음. `.html` 은 Finder 기본 핸들러가 Safari 로 고정되어 있고, 어떤 macOS 버전에서도 예측 가능하게 동작.

추가 이점:
- 사용자에게 **시각 피드백** — 빈 화면 대신 안내 페이지를 봄
- 자동 호출 실패해도 **수동 버튼 제공**
- 여러 URL scheme fallback 을 JS 로 추가할 여지

## 변경 없음 (beta.3 과 동일)

Ad-hoc 서명, `hardenedRuntime: false`, 모든 기능 (SSH 원격, 이미지 IPC, 한국어 UI 등).
