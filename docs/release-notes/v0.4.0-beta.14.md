# v0.4.0-beta.14 — project tabs and calmer wiki

> 2026-05-13 · macOS prerelease (free ZIP, ad-hoc signed)

beta.14 makes Markwand feel more like a multi-project desktop workspace. You can keep several projects open as tabs, return to each project's previous reading state, and read Project Wiki through a simpler first screen that starts with the next useful action.

## Added

- **Project tabs** — opening a project now keeps it in a tab bar, so switching projects no longer requires returning to the project list.
- **Per-project view sessions** — each tab remembers the selected document, wiki/document mode, and scroll position.
- **Closed tab recovery** — recently closed project tabs can be reopened with their saved view session.
- **Tab controls** — tabs support close, drag reorder, overflow scroll buttons, keyboard navigation, and a right-click menu for closing one tab, other tabs, or tabs to the right.
- **Tab shortcuts** — `Command+1...9` switches tabs, `Command+W` closes the current project tab, `Command+Shift+T` reopens a closed tab, and `Command+Shift+[` / `Command+Shift+]` moves between tabs.

## Changed

- **Project Wiki first screen** — the wiki now starts with the next action, quick summary, and AI-ready tasks instead of exposing every diagnostic panel at once.
- **Progressive wiki details** — Git flow, link graph, document roles, document debt, decisions, and metadata facets are tucked behind a clear "See more" section.
- **Simpler wiki language** — Korean and English labels remove product jargon such as "Project Brief", "Risk Board", and "Doc Debt Radar" from the main UI.
- **Active tab clarity** — the selected tab has a stronger border, top accent, and contrast from inactive tabs.

## Fixed

- `Command+W` closes a project tab when tabs are open, but does nothing inside the app when there are no project tabs to close.
- Project tabs are pruned after a workspace rescan so missing projects do not stay open as stale tabs.
- Project Wiki section navigation opens collapsed details before scrolling to a section inside them.
- Slow document reads no longer restore stale content into the wrong project tab.
