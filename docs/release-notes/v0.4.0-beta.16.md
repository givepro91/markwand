# v0.4.0-beta.16 — refresh and highlight recovery

> 2026-05-14 · macOS prerelease (free ZIP, ad-hoc signed)

beta.16 fixes two state bugs that made Markwand feel unreliable in real projects: the project refresh button could fail to visibly reload newly added folders/files, and text highlighting could show the toolbar but fail to create the highlight after focus moved to the toolbar button.

## Fixed

- **Project file-tree refresh** — refresh now ignores stale scan chunks/results and keeps loading state tied to the latest scan generation.
- **Folder/file additions after refresh** — document buckets are updated immutably so React selectors re-render when newly scanned files or folders arrive.
- **Watcher/scan races** — add/unlink/change events that arrive during a scan no longer let older scan results overwrite fresher project state.
- **Main-process scan cache races** — older in-flight scans can no longer overwrite the newest cached document list.
- **Highlight creation** — selection is converted to a static annotation selector before the toolbar steals focus, so Chromium/Electron selection collapse no longer empties the highlight action.
- **Highlight toolbar click path** — toolbar mousedown no longer prevents the button's default click path.
