# v0.4.0-beta.12 — viewer polish and trust reset

> 2026-05-12 · macOS prerelease (free ZIP, ad-hoc signed)

beta.12 turns the recent dogfooding feedback into a steadier everyday viewer: image zoom behaves like a canvas, file trees stay in sync, document actions are smaller and clearer, and low-confidence warning surfaces are hidden until they can earn user trust.

## Added

- **Image viewer controls** — zoom buttons, wheel zoom, keyboard shortcuts, double-click fit/100%, and grab-pan for inspecting drawings and screenshots.
- **Current document actions** — reveal the current file location, open the actual current file in the preferred app, copy the title, copy the path, and copy raw Markdown source.
- **Update awareness** — the header now checks for newer GitHub prereleases at startup and periodically while the app stays open, then shows a visible update badge instead of relying on users to click blindly.
- **File tree refresh guard** — the sidebar tree now receives watcher updates for create/delete/change events and also exposes a local tree refresh button.
- **Electron layout smoke test** — `pnpm smoke:layout` launches a real Electron window with fixture data and verifies 1400/1440/1600px document layouts with the TOC rail open.

## Changed

- **Reference warnings are de-emphasized** — broken-link and code-reference warnings are no longer shown as a default product surface because false positives were eroding confidence.
- **File creation controls are hidden** — new Markdown, new folder, rename, and trash remain in code but are not exposed until the app has a real editing workflow.
- **Document header is compact** — the sticky reading bar now sits flush at the top, uses less vertical space, and keeps actions from covering body text.
- **TOC rail is narrower** — at 1440px the table-of-contents rail is about 259px, preserving the central reading pane.
- **Markdown readability pass** — typography, colors, code/table surfaces, and contrast were tuned toward a calmer reader-first style.

## Fixed

- Image fit/actual-size behavior no longer crops unintentionally, and the removed fill mode no longer offers a misleading crop-only path.
- Wheel zooming an image no longer scrolls the whole document chrome out of view.
- Right-side document tool panels can scroll when content is long.
- Compact window layouts no longer leave the sticky document header floating below an unnecessary top gap.
- Update checks no longer crash when the Electron update API is unavailable in the free ZIP build.
