## 🛠 변경 상세

### A. 글로벌 새로고침 + 버전 노출

- **Sidebar 우측 영역**: `[🔄 회전 가능 IconButton] [v0.4.0-beta.4] [☀☾⊙ 테마] [⚙ 설정]` 순서. workspace 미선택 시 새로고침 disabled
- **⌘R 단축키**: 입력 필드 포함 어디서나 동작 (`useGlobalHotkey`). `preventDefault` 로 Electron default reload 가로채기
- **AllProjectsView 기존 새로고침 IconButton**: `title="새로고침 (⌘R)"` 호버 툴팁 추가 → 마우스 사용자 인지 강화
- **버전 텍스트**: `electron.vite.config.ts` 의 `define: __APP_VERSION__` 으로 build-time 주입. IPC 불필요. `userSelect: 'text'` 로 사용자가 복사해 보고에 붙여넣기 가능
- **회전 애니메이션**: `prefers-reduced-motion` 존중

### B. 풀스크린 오버레이 깜빡임 제거

기존: 새로고침 누르면 `setProjectsLoading(true)` + `setDocCountProgress({done:0, total:0})` 가 다시 발화 → 풀스크린 "워크스페이스 분석 중…" 모달이 깜빡 등장.

이후:
- `shouldShowInitialOverlay` 헬퍼가 `projectsCount === 0` 일 때만 true 반환
- `setActiveWorkspaceId` 가 ws id 변경 시 projects/projectsError 자동 비움 → 워크스페이스 전환은 첫 진입과 동일 처리
- 명시 새로고침은 풀스크린 모달 안 뜸. Sidebar 새로고침 버튼 회전 + AllProjectsView 헤더 inline progress bar 만으로 진행 신호 충분

### C. 새로고침 시 데이터 재로드

기존: workspace.refresh IPC 만 호출 → 프로젝트 목록은 갱신되어도 (1) 현재 프로젝트의 docs 목록, (2) 열린 마크다운 본문이 stale.

이후:
- `useReloadOnRefresh` hook — `store.refreshKey` 변경 시 reload 콜백 호출. mount 첫 발화 무시 + ref 패턴으로 stale closure 차단 + unmount 후 ghost handler 차단
- `useDocs` deps 에 refreshKey → 현 프로젝트 docs 재스캔
- `ProjectView` 가 selectedDoc 이 있으면 `readDoc(path)` 다시 호출 → MarkdownViewer 갱신

### D. 이미지 cache busting

기존: 외부에서 이미지 변경해도 `<img src="app://local/...">` 가 Chromium cache hit → 새로고침해도 stale.

이후:
- `buildLocalImageSrc(absPath, refreshKey)` → `app://local/<encoded>?r=<key>`
- `ImageViewer` (단독) + `MarkdownViewer` 인라인 이미지 모두 적용
- SSH 이미지 (blob URL) 은 useEffect deps 에 refreshKey → IPC 재요청 → 새 blob
- 외부 URL (http/https/data/blob) 은 토큰 부착 안 함 (서버 영향 차단)
- main 측 `app://` protocol handler 는 `url.pathname` 만 사용 → 쿼리는 파일 해석에 영향 없음

### E. FileTree 스크롤 보존

기존: 새로고침 시 `useDocs.scanDocs` 가 현 프로젝트 docs 즉시 비웠다 다시 채움 → react-arborist `<Tree>` 가 빈 데이터 분기에 빠져 unmount → 첫 chunk 도착 시 다시 mount → 스크롤 0 으로 리셋.

이후:
- `appendDocs` path-dedup. projectId 별 보조 인덱스로 같은 path 는 replace, 새 path 는 push
- `scanDocs` 가 시작 시 docs 안 비움. chunk 도착 시 dedup append, `.then(result)` 에서 ground truth 와 비교해 stale path 만 일괄 제거
- `<Tree>` 가 항상 mount 상태 유지 → 스크롤 위치 유지

### F. RecentDocsPanel "+N개 더" 클릭 가능

기존: ProjectView 좌측 "최근 7일" 패널에 `MAX_ITEMS=10` 만 보여주고 나머지는 "+140개 더" 정적 텍스트 → 클릭 불가, 확인 방법 없음.

이후:
- `onSeeMore` prop 추가. button 으로 렌더 + hover bg + aria-label/title
- 클릭 시 `setViewMode('inbox')` → InboxView 로 이동해 모든 최근 문서 그룹 표시
- prop 미지정 시 정적 텍스트 fallback (하위 호환)
