# v0.4.0-beta.15 — markdown callouts and table polish

> 2026-05-14 · macOS prerelease (free ZIP, ad-hoc signed)

beta.15 improves the everyday state-document reading loop. GFM Alerts now render as real callouts instead of noisy blockquotes, markdown tables avoid unnecessary horizontal scrolling for common 3-4 column status tables, and Codex UI checks now have a project rule to avoid full desktop screenshots.

## Added

- **GFM Alert callouts** — `> [!NOTE]`, `TIP`, `IMPORTANT`, `WARNING`, and `CAUTION` render as distinct callout boxes with a title, icon, theme-aware color, and hidden raw marker.
- **Safe details blocks** — standalone `<details><summary>...</summary>...</details>` blocks render as native collapsible sections while preserving markdown rendering inside the body.
- **Same-document hash navigation** — `#heading` links scroll to their heading target and briefly highlight it.
- **Codex app capture rule** — `AGENTS.md` now requires app-window/DOM/CDP-based UI verification instead of full macOS desktop screenshots.

## Changed

- **Markdown table layout** — common low-column tables now fit the reader width with wrapping instead of immediately exposing a horizontal scrollbar.
- **Dense table fallback** — 5+ column tables still keep horizontal overflow containment for matrix-like content.
- **Reader width** — the markdown surface widens slightly from 840px to 920px while preserving 16px prose text.
- **Task lists and footnotes** — checkbox and footnote styling now reads more like first-class GFM output.

## Fixed

- Mermaid render failures now show a visible error state with the original diagram source instead of leaving a blank or raw red `<pre>`.
- Mermaid loader failures are caught and displayed in the same error state.
- Details-like markup inside fenced code blocks is no longer misinterpreted as a real collapsible section.
- Malformed hash links no longer throw during click handling.
