# v0.4.0-beta.17 — scoped document analysis

> 2026-05-16 · macOS prerelease (free ZIP, ad-hoc signed)

beta.17 fixes a real-world slowdown where document counts in a local workspace could be blocked by unrelated SSH workspaces registered earlier in the app. Project analysis now carries the owning workspace through the renderer, preload, IPC validators, and main-process scan lookup, so a project is counted and scanned only inside the workspace it belongs to.

## Fixed

- **All Projects document counts** — project cards now pass the active workspace id to `project:get-doc-count`, avoiding cross-workspace probing.
- **Project document scans** — `project:scan-docs` accepts the same workspace hint, so file-tree refreshes and project entry scans stay scoped to the selected project workspace.
- **Unreachable SSH isolation** — an offline or unreachable SSH workspace no longer slows down document counts for a healthy local workspace.
- **Workspace/project cache mapping** — project-to-workspace mapping is recorded as soon as a workspace scan completes, keeping later doc scans deterministic.
- **Document scroll restoration** — project document scroll restore now runs across two animation frames, preventing restored content from briefly overlapping the sticky document toolbar after layout settles.
