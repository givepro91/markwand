# v0.4.0-beta.13 — instant quit and viewer clarity

> 2026-05-12 · macOS prerelease (free ZIP, ad-hoc signed)

beta.13 is a dogfooding hotfix for the everyday viewer loop: quitting the app now feels immediate, document actions read more clearly, and the dev shell no longer leaves Electron feeling stuck after a normal quit.

## Changed

- **Fast quit path** — quit now hides windows synchronously, starts SSH/watch cleanup as best-effort work, and exits the Electron PID immediately instead of waiting on native teardown.
- **Dev wrapper shutdown** — `pnpm dev` now receives a fast-quit signal from the app and exits with the Electron child instead of leaving the shell session feeling alive.
- **DevTools opt-in** — development runs no longer auto-open DevTools unless `MD_VIEWER_DEBUG=1` or `MD_VIEWER_DEBUG=true` is set.
- **Document action labels** — the current document bar and TOC button labels now describe the actual state/action more directly in Korean and English.
- **Compact opener label** — the narrow project opener uses a shorter, more scannable VS Code label.

## Fixed

- Closing via the app quit event no longer leaves the main Electron process alive for several seconds after helper processes have exited.
- Quit cleanup now handles synchronous cleanup failures through the same logging path as async cleanup failures.
- The table-of-contents button no longer reads like a static label when the panel is already open.
