# v0.4.0-beta.2 — Critical Hotfix (스캔 회귀 · 무한 렌더 · SSH drift)

> 2026-04-25 · macOS prerelease (unsigned DMG)

---

## ⚠️ v0.3.0-beta.9 / v0.4.0-beta.1 사용자는 즉시 업그레이드 권장

이전 두 릴리스에는 다음 4종 회귀가 있었습니다:

1. **로컬 워크스페이스 첫 스캔이 4500배 느려짐** — `~/develop` 같은 큰 트리에서 "워크스페이스 분석 중…" 에 분 단위 멈춤
2. **`Maximum update depth exceeded`** — 필터바 마운트 시 무한 렌더 → 화면 깨짐
3. **SSH 워크스페이스 drift 검증 실패** — `ENOENT lstat` 오류 stderr 도배
4. **dev DevTools UI 노이즈** — Autofill / VE language-mismatch 무해 에러 출력

이 4건이 모두 한 릴리스(beta.2)에서 정리됐습니다. 이전 DMG 는 사용 자제를 권장합니다.

## 🛠 핵심 수정

### 1. 로컬 스캔 4500배 회귀 fix · `src/main/services/watcher.ts` · `src/main/index.ts`

**원인**: v0.3.0-beta.9 에서 chokidar watcher 가 `~/develop` 수준 워크스페이스도 안전하다고 판단해 재활성화됐는데, 실측 결과 큰 트리(swk: 15,279 디렉토리)에서 chokidar 의 startup walk 가 libuv 4-스레드풀을 점거 → 동시에 도는 첫 `scanProjects` 의 `fs.access` 호출이 4500배 느려짐.

**측정값**:
| 워크스페이스 | beta.1 | beta.2 | 배수 |
|---|---:|---:|---:|
| swk (15,279 폴더) | **131,486 ms** | **148 ms** | **888배** |
| 다른 로컬 / SSH | 정상 | 정상 | – |

**수정**:
- `startWatcher` 호출을 `initializeApp` 직후 → **5s defer** (첫 IPC scan 이 chokidar 와 경쟁 안 함)
- chokidar `depth: 10 → 4` (프로젝트 레벨 변경 감지에 충분, 깊은 트리 walk 부담 ↓)

### 2. `Maximum update depth exceeded` fix · `src/renderer/hooks/useDocs.ts`

**원인**: v0.4 S4 에서 추가된 `useFrontmatterIndex` selector 가 매 호출 새 객체+새 배열 반환 → Zustand 가 참조 불일치로 매 store 변경마다 강제 re-render → React 18 `getSnapshot should be cached` 감지 → FilterBar 에서 무한 루프 발현.

**수정**:
- `useFrontmatterIndex` — 원시 Set 참조 구독 + 파생 Array 는 `useMemo` 캐시
- `useDocsOf` — `?? []` 매 호출 새 배열 → 모듈 상수 `EMPTY_DOCS` 로 참조 안정화

### 3. SSH 워크스페이스 drift ENOENT 도배 fix · `src/main/ipc/drift.ts`

**원인**: `drift:verify` IPC 핸들러가 `localTransport` 하드코딩 → SSH 경로(`/home/ubuntu/...`)를 로컬 `fs.lstat` 로 호출 → ENOENT throw → Electron IPC 에러 로깅 도배.

**수정**:
- `localTransport.fs.*` 4곳 → `transport.fs.*` (workspace 역매핑 경유)
- `resolveTransportForPath(docPath, workspaces)` 로 SSH/로컬 자동 분기 (이미 `fs:read-doc` 가 사용하는 헬퍼와 동일 패턴)
- `assertInWorkspace({posix: isSsh})` 로 POSIX 경로 검증

### 4. dev DevTools UI 노이즈 정리 · `scripts/dev.mjs` · `package.json`

`pnpm dev` 터미널에 도배되던 Electron 33 + 내장 DevTools UI 의 무해 노이즈(Autofill CDP 미구현 · VE language-mismatch) 를 stderr 필터 wrapper 로 제거. raw 출력 필요 시 `pnpm dev:raw` 우회.
