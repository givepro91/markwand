# v0.4.0-beta.11 — project openers and reference accuracy

> 2026-05-03 · macOS prerelease (free ZIP, ad-hoc signed)

beta.11 focuses on real-world dogfooding fixes from large local and SSH workspaces: opening projects in the tools people actually use, reducing false broken-link noise, and preventing recurring overlay clipping regressions.

## Added

- **Open project with installed apps** — local projects can now be opened from the sidebar in VS Code, Finder, Terminal, iTerm, Ghostty, Xcode, IntelliJ IDEA, or Cursor when detected.
- **Default project opener** — users can save their preferred app, and the sidebar button uses that choice automatically.
- **SSH-safe behavior** — remote SSH projects clearly disable local app openers instead of attempting to open unusable local paths.
- **Plain code path references** — Trust & Drift now understands plain prose paths such as `src/components/toast-provider.tsx`, not only backticked or `@/...` references.

## Fixed

- Settings and project opener dialogs now render through top-level layers so headers and sidebars cannot clip them.
- App detection failure now falls back only to core choices: VS Code, Terminal, and Finder.
- iTerm launching uses the actual macOS AppleScript application name (`iTerm`), avoiding false failures on systems where the UI label is iTerm2.
- Ghostty no longer uses `-e /bin/zsh`, preventing the extra `/bin` tab regression.
- Link Graph cards now contain long document names instead of overflowing into neighboring columns.
- Drift verification can resolve basename-only code hints with a bounded workspace lookup, while avoiding expensive dependency/build folders.
