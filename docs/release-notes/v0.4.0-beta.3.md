## ✨ 새로 추가된 기능 — 텍스트 하이라이트

마크다운 문서를 읽으면서 **중요한 부분에 노란 형광펜**을 칠할 수 있습니다.

### 사용법

1. 본문에서 원하는 텍스트를 **드래그로 선택** → 선택 위에 🖍 **하이라이트** 버튼이 뜸 → 클릭
2. 노란 하이라이트가 텍스트에 적용되고, **앱을 다시 켜도 그대로 복원**됩니다
3. 하이라이트된 텍스트를 **클릭** → ✕ **제거** 버튼으로 삭제

### 데이터 저장 방식

- 하이라이트는 원본 `.md` 옆에 `<문서경로>.anno.json` sidecar 파일로 저장
- 원본 마크다운은 **건드리지 않음** — 다른 에디터/뷰어와 호환
- JSON schema: `{ version: 1, annotations: [{ id, selector: { type: 'TextQuote', exact, prefix?, suffix? }, positionFallback?, color: 'yellow', createdAt }] }`
- 본문이 약간 수정되어도 **fuzzy 매칭으로 위치 자동 복원** (text-quote anchor + diff-match-patch). 매칭 실패 시 상단에 "위치를 찾지 못한 하이라이트 N개" 뱃지

### 현재 한계 (v0.5 에서 확장 예정)

- **노랑 1색만** — 다색 팔레트는 v0.5
- **로컬 .md only** — 원격 SSH 워크스페이스의 문서에서는 toolbar 가 disabled 상태로 표시됩니다 (v0.5 지원 예정)
- **note 텍스트 popover 없음** — 하이라이트만, 메모 기능은 v0.5 이후
- annotation 검색, `.md` footnote 내보내기 — v0.5 이후

## 🛠 구현 핵심

### Anchor 라이브러리

- `dom-anchor-text-quote` 4.0.2 + `dom-anchor-text-position` 5.0.0 (`pnpm overrides` 로 5.0.0 강제 — quote 의 `^4` 의존성이 jsdom 25 Range API 와 비호환)
- `Range → { exact, prefix, suffix, start, end }` 변환 + 본문 변경 후 fuzzy 복원

### CSS Custom Highlight API

- `::highlight(markwand-annotation-highlight)` 단일 규칙 — DOM 노드 추가 없이 Range 기반 색칠
- `rgba(255, 213, 79, 0.8)` + `color: #1f2328` — 라이트/다크 모드 모두 ≥ WCAG AA 대비
- pointer-events 미수신은 `caretRangeFromPoint(x,y)` 역조회로 hit detection 보완

### IPC + 보안

- `annotation:load` / `annotation:save` 핸들러 (`src/main/ipc/annotation.ts`)
- main 측 zod schema (`AnnotationFileSchema`) + `assertInWorkspace` + 2MB 상한 + atomic write (tmp → rename) + 빈 배열 시 sidecar 자동 삭제
- SSH workspace 는 `ANNOTATION_SSH_UNSUPPORTED` throw (UI 측 disabled 와 이중 가드)

### UX 가드

- **드래그 도중에는 toolbar 안 뜸** — `mousedown` → `isDragging=true` + toolbar hide. `selectionchange` 는 drag 중 무시. `mouseup` 시점에 한 번만 컴퓨트 (1글자 미세 selection / 공백만 selection 차단)
- **doc 전환 시 동일 단어가 잘못된 위치에 잠깐 highlight 되는 race 차단** — `prevDocPathRef` 로 docPath 변경 first cycle 매칭 skip + 즉시 `CSS.highlights.delete`
- **toolbar ✕ click 무동작 회귀 차단** — `[data-annotation-toolbar]` closest + `stopPropagation` 으로 root click 핸들러와 격리
- 키보드: toolbar 출현 시 첫 버튼 autoFocus, ESC 로 dismiss
- 저장 실패 시 `toast.error("하이라이트 저장에 실패했어요")` (Silent fail 차단)
- 2000자 초과 선택은 `toast.info("선택이 너무 길어요")` 안내
