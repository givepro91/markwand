# Markwand v0.3.0-beta.2 — 설치 경험 개선 (Ad-hoc 서명)

**배포일**: 2026-04-22
**배포 채널**: 내부 베타 (unsigned dmg + ad-hoc signing)
**이전 릴리스**: v0.3.0-beta.1 (2026-04-22)

기능 추가 없음. **설치 시 "앱이 손상되었습니다" 경고 회피** 가 목적인 릴리스입니다. 기능은 beta.1 과 동일합니다.

---

## 🚀 설치 빠른 경로 (macOS Sequoia 15 / Tahoe 26 기준)

1. 아래 [Assets](#assets) 에서 본인 Mac 기종에 맞는 DMG 다운로드 (Apple Silicon 은 `arm64`)
2. DMG 더블클릭 → `Markwand` 를 **Applications** 폴더로 드래그
3. Applications 에서 Markwand 더블클릭 → "악성 코드 확인 불가" 경고 → **완료** 클릭
4. 👉 **[시스템 설정 → 개인정보 보호 및 보안 열기](x-apple.systempreferences:com.apple.settings.PrivacySecurity.extension)** (이 링크 클릭)
5. 스크롤 → "`Markwand.app` 이(가) 차단되었습니다" 옆 **그래도 열기** 클릭 → Touch ID / 암호
6. Markwand 다시 더블클릭 → **열기** 확인

이후부터는 더블클릭만으로 바로 실행됩니다.

---

## 바뀐 것

### 🖱 설치 과정에서 터미널 명령 불필요

**beta.1 (이전)**:
- macOS Sequoia 에서 "앱이 손상되었습니다" 경고 → 실행 불가
- 터미널에서 `xattr -cr /Applications/Markwand.app` 필수

**beta.2 (현재)**:
- DMG 내부 앱에 **ad-hoc 서명** 자동 삽입 (`afterPack` hook 에서 `codesign --force --deep --sign -`)
- Identifier `tech.spacewalk.markwand` 로 고정 → Electron 기본 identifier 가 Gatekeeper 오판을 유발하던 문제 해소
- 경고 메시지가 **"손상되었습니다"** → **"악성 코드가 없음을 확인할 수 없습니다"** 로 완화
- 첫 실행 시 터미널 명령 필요 없음 — **시스템 설정 → 개인정보 보호 및 보안 → 그래도 열기** 로 통과 (macOS Sequoia 15+ 기준) 또는 **우클릭 → 열기** (Sonoma 14 이하)

### 🛠 빌드 구성 조정

- `hardenedRuntime: true` → `false` (공증 없으면 오히려 해롭기만 한 옵션)
- `entitlements` · `entitlementsInherit` 제거 (hardened 미사용 시 불필요)
- `build/afterPack.js` 신규 — 패키징 직후 ad-hoc 서명 스크립트
- `identity: null` 명시 (electron-builder 공식 코드사이닝 건너뛰기)
