## 🛠 Hotfix — 첫 SSH 접속 시 인증서 신뢰 모달이 안 뜨던 문제

### 증상

워크스페이스를 **한 번도 추가하지 않은 상태**(빈 상태 화면)에서 `+ 원격 서버(SSH)` 버튼으로 첫 SSH 접속을 시도하면,

- 서버 호스트 키 신뢰 확인(TOFU) 모달이 **노출되지 않음**
- **20초 뒤 자동으로 연결 실패** — 타임아웃처럼 보이는 오류
- 터미널/VSCode 로는 같은 PC 에서 정상 접속됨 → 네트워크/SSH 키/보안그룹 이슈 아님

이미 워크스페이스가 1개 이상 등록된 PC(기존 사용자)에서는 정상 동작하므로, **첫 사용자** 에게만 발생하는 블로커였습니다.

### 원인

renderer 의 empty-state 렌더 분기에 TOFU 모달 컴포넌트(`<SshHostKeyPrompt />`)가 mount 되지 않아, `ssh:host-key-prompt` IPC 이벤트를 수신할 listener 자체가 등록되지 않았습니다. main 프로세스가 host key 확인 요청을 보내도 renderer 쪽에 도달하지 못하고, bridge 의 20초 타임아웃(`DEFAULT_PROMPT_TIMEOUT_MS`)이 걸리면 자동 거부되어 연결 실패로 귀결.

### 수정

`src/renderer/App.tsx` empty-state 분기에도 `<SshHostKeyPrompt />` 를 mount. 워크스페이스가 있을 때(두 번째 분기)와 동일하게 IPC listener 가 상시 등록되도록 정합화.

- 2 파일 변경 (`package.json` 버전 bump + `src/renderer/App.tsx` 2줄 추가)
- 타입체크 PASS, 독립 Evaluator PASS (4/4 항목)
- 중복 mount · 메모리 누수 위험 없음 (React early-return 분기 구조상 두 인스턴스 동시 mount 불가)
